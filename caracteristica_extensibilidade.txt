# Extensibilidade

A característica extensibilidade se refere à capacidade de um sistema de ser
estendido para incluir novas funcionalidades sem afetar negativamente as
funcionalidades existentes. Isso pode ser alcançado através de estruturas de
código flexíveis e escaláveis, baixo acoplamento entre as partes do código e
uma arquitetura bem projetada. Isso permite que o sistema se adapte às mudanças
de requisitos e evite a necessidade de grandes mudanças no código existente.

A característica extensibilidade tem uma relação direta com alguns maus-cheiros
de código, como "Função que faz muito" e "Classe complexa". A extensibilidade é
importante para evitar esses maus-cheiros, pois eles podem tornar o código
difícil de entender e manter.

Uma operação de refatoração que pode ser usada para aumentar a extensibilidade
é a "extração de classe", que consiste em dividir uma classe grande e complexa
em mais de uma classe menor e mais simples, cada uma com uma responsabilidade
específica. Isso pode tornar o código mais fácil de entender e manter, além de
aumentar a reutilização de código e permitir a adição de novas funcionalidades
sem afetar as existentes.
